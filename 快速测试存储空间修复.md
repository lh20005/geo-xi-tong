# 快速测试存储空间修复

## 1. 启动系统

```bash
# 启动后端
npm run server:dev

# 启动前端（新终端）
npm run client:dev
```

## 2. 登录测试账号

使用以下任一账号登录：
- **lzc2005** (管理员，1GB 配额)
- **testuser2** (专业版用户，1GB 配额)
- **test** (普通用户，10MB 配额)

## 3. 测试用户中心

1. 访问 **用户中心** → **存储空间** 标签页
2. 验证显示：
   - ✅ 配额显示正确（如：1 GB）
   - ✅ 已使用空间显示正确
   - ✅ 剩余空间显示正确
   - ✅ 使用率百分比正确

## 4. 测试企业图库上传

1. 访问 **企业图库**
2. 创建新相册或选择现有相册
3. 上传图片（建议 1-5MB）
4. 验证：
   - ✅ 上传成功
   - ✅ 图片显示在相册中
   - ✅ 返回用户中心，存储空间已更新

## 5. 测试知识库上传

1. 访问 **企业知识库**
2. 创建新知识库或选择现有知识库
3. 上传文档（PDF、Word、TXT）
4. 验证：
   - ✅ 上传成功
   - ✅ 文档显示在知识库中
   - ✅ 返回用户中心，存储空间已更新

## 6. 测试配额限制

使用 **test** 账号（10MB 配额）：
1. 尝试上传 15MB 的文件
2. 验证：
   - ✅ 显示错误提示
   - ✅ 提示信息准确（空间不足）
