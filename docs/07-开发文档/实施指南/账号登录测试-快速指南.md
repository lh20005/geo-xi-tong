# 账号登录测试 - 快速指南

## 🎯 功能说明

在Windows端平台管理页面的账号列表中，每个账号后面都有一个"测试登录"按钮。点击后会：
1. 自动打开浏览器
2. 使用保存的Cookie登录平台
3. 导航到平台主页
4. 由用户自己查看登录状态
5. 用户自己决定何时关闭浏览器

## 🚀 使用方法

### 方法1：使用界面（推荐）

```
1. 打开Windows登录管理器
2. 进入"平台管理"页面
3. 在账号列表中找到要测试的账号
4. 点击该账号行的"测试登录"按钮
5. 等待浏览器打开
6. 查看登录状态
7. 查看完毕后手动关闭浏览器
```

### 方法2：使用脚本

```bash
# 测试指定账号（替换123为实际账号ID）
./test-account-login.sh 123
```

## 📊 操作说明

### ✅ 成功打开
- 消息：浏览器已打开，请查看登录状态。查看完毕后请手动关闭浏览器窗口。
- 说明：浏览器已打开，Cookie已设置，请自己查看

### ❌ 打开失败
- 可能的原因：
  - 账号不存在
  - Cookie未保存
  - 浏览器启动失败
  - 网络连接问题

## 🎯 使用场景

1. **快速验证** - 点击按钮即可查看登录状态
2. **发布前验证** - 发布文章前测试账号
3. **问题排查** - 发布失败后测试账号
4. **日常检查** - 定期检查账号状态
5. **平台操作** - 可以在打开的浏览器中进行任何平台操作

## 📝 注意事项

- 浏览器不会自动关闭，由用户自己决定何时关闭
- 可以同时打开多个浏览器窗口测试不同账号
- Cookie可能会过期，如发现未登录需要重新登录
- 需要安装Chrome浏览器
- 查看完毕后请手动关闭浏览器窗口
