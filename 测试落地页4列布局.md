### 5. 测试响应式

#### 方法1：调整浏览器窗口
1. 打开浏览器开发者工具（F12）
2. 点击"切换设备工具栏"图标（Ctrl+Shift+M）
3. 选择不同的设备预设：
   - iPhone SE（375px）→ 1列
   - iPad（768px）→ 3列
   - iPad Pro（1024px）→ 3列
   - 笔记本（1440px）→ 4列
   - 桌面（1920px）→ 4列

#### 方法2：手动调整宽度
1. 拖动浏览器窗口边缘
2. 观察布局变化：
   - 宽度 < 640px：1列
   - 640px ≤ 宽度 < 1024px：2列
   - 1024px ≤ 宽度 < 1280px：3列
   - 宽度 ≥ 1280px：4列

## 常见问题

### Q1: 卡片高度不一致？
**A**: 检查以下几点：
1. 确保使用了 `flex flex-col` 布局
2. 确保上半部分有 `flex-grow`
3. 确保下半部分有 `mt-auto`
4. 描述文本使用了 `line-clamp-2`

### Q2: 响应式不生效？
**A**: 
1. 清除浏览器缓存（Ctrl+Shift+R）
2. 检查 Tailwind CSS 是否正确编译
3. 确认断点类名正确：`sm:` `lg:` `xl:`

### Q3: 卡片显示不全？
**A**: 
1. 检查容器宽度：`max-w-7xl`
2. 检查间距：`gap-6`
3. 确保没有固定宽度限制

### Q4: 移动端显示异常？
**A**: 
1. 确认使用了 `grid-cols-1` 作为基础
2. 检查移动端视口设置
3. 测试不同移动设备
