# 🚀 启动服务指南

## 完整启动流程

### 方法1：使用两个终端窗口（推荐）

#### 终端1 - 启动后端

```bash
cd server
npm run dev
```

**确认看到**：
```
✅ 加密服务初始化成功
🚀 服务器运行在 http://localhost:3000
```

#### 终端2 - 启动前端

```bash
cd client
npm run dev
```

**确认看到**：
```
VITE v5.x.x  ready in xxx ms

➜  Local:   http://localhost:5173/
➜  Network: use --host to expose
➜  press h + enter to show help
```

### 方法2：使用启动脚本

如果有 `start.command` 文件：

```bash
./start.command
```

## 📋 访问地址

启动成功后：

- **前端页面**：http://localhost:5173
- **后端API**：http://localhost:3000/api

## ✅ 验证服务是否运行

### 检查端口

```bash
# 检查后端（3000端口）
lsof -i :3000

# 检查前端（5173端口）
lsof -i :5173
```

### 测试访问

```bash
# 测试后端API
curl http://localhost:3000/api/health

# 测试前端（在浏览器中打开）
open http://localhost:5173
```

## 🐛 常见问题

### 问题1：端口被占用

**错误信息**：
```
Error: listen EADDRINUSE: address already in use :::3000
```

**解决方案**：
```bash
# 查找占用端口的进程
lsof -i :3000

# 杀死进程
kill -9 <PID>

# 或者修改端口
# 在 .env 文件中修改 PORT=3001
```

### 问题2：依赖未安装

**错误信息**：
```
Error: Cannot find module 'xxx'
```

**解决方案**：
```bash
# 安装后端依赖
cd server
npm install

# 安装前端依赖
cd client
npm install
```

### 问题3：前端无法访问

**检查**：
1. 前端服务是否启动
2. 端口5173是否被占用
3. 浏览器是否输入了正确的地址

**解决方案**：
```bash
# 重启前端服务
cd client
# 按 Ctrl+C 停止
npm run dev
```

### 问题4：后端API无法访问

**检查**：
1. 后端服务是否启动
2. 端口3000是否被占用
3. 数据库是否连接成功

**解决方案**：
```bash
# 重启后端服务
cd server
# 按 Ctrl+C 停止
npm run dev
```

## 💡 提示

- **保持两个终端窗口打开**，这样可以同时看到前端和后端的日志
- **不要关闭终端**，否则服务会停止
- **如果修改了代码**，需要重启相应的服务

## 🔄 重启服务

如果需要重启：

```bash
# 在运行服务的终端中按 Ctrl+C 停止
# 然后重新运行启动命令

# 后端
cd server
npm run dev

# 前端
cd client
npm run dev
```
