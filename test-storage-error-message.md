# 测试存储空间错误消息

## 测试步骤

1. 确保后端已重新编译并重启
2. 登录一个存储空间已满的用户
3. 尝试上传图片或文档
4. 检查错误消息

## 预期结果

应该看到一个模态框，标题为"存储空间不足"，内容包括：
- 主要错误信息："存储空间不足，无法上传图片。请升级套餐以获取更多存储空间。"
- 详细原因（如：当前已使用 XX MB / XX MB）
- 按钮："前往个人中心"

## 调试步骤

如果仍然显示"上传失败"而不是"存储空间不足"：

1. 检查后端是否已重启
2. 检查浏览器控制台的网络请求，查看实际返回的错误
3. 检查前端是否已刷新（清除缓存）

## 快速测试命令

```bash
# 1. 重新编译后端
cd server && npm run build

# 2. 重启后端服务
# 停止当前运行的后端，然后重新启动

# 3. 清除浏览器缓存并刷新前端
```
