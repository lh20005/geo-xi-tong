## 在Kiro中提交代码的步骤

### 1. 打开源代码管理面板
- 点击左侧边栏的"源代码管理"图标（分支图标）
- 或使用快捷键 `Ctrl+Shift+G` (Windows/Linux) 或 `Cmd+Shift+G` (Mac)

### 2. 查看更改
- 你会看到所有修改过的文件列表
- 文件旁边有字母标记：
  - `M` = Modified (修改)
  - `A` = Added (新增)
  - `D` = Deleted (删除)
  - `U` = Untracked (未跟踪)

### 3. 暂存更改
**方法A: 暂存所有文件**
- 点击"更改"标题旁边的 `+` 号
- 或点击"全部暂存"按钮

**方法B: 暂存单个文件**
- 将鼠标悬停在文件上
- 点击文件旁边的 `+` 号

### 4. 编写提交信息
在顶部的消息框中输入提交信息，例如：
```
feat: 添加新功能

- 功能描述1
- 功能描述2
```

**提交信息规范**：
- `feat:` - 新功能
- `fix:` - 修复bug
- `docs:` - 文档更新
- `style:` - 代码格式调整
- `refactor:` - 重构代码
- `test:` - 测试相关
- `chore:` - 构建/工具相关

### 5. 提交
- 点击 `✓ 提交` 按钮
- 或使用快捷键 `Cmd+Enter` (Mac) / `Ctrl+Enter` (Windows)

### 6. 推送到GitHub
**方法A: 使用推送按钮**
- 点击 `...` 菜单（三个点）
- 选择"推送"或"Push"

**方法B: 使用状态栏**
- 查看底部状态栏
- 点击"推送"图标（向上箭头）

**方法C: 使用命令面板**
- 按 `Cmd+Shift+P` (Mac) 或 `Ctrl+Shift+P` (Windows)
- 输入"Git: Push"
- 按回车

## 常见问题

### Q1: 点击提交后没有反应？
**A**: 检查是否已暂存更改（文件要从"更改"区移到"暂存的更改"区）

### Q2: 提交成功但GitHub上看不到？
**A**: 需要推送！提交只是保存到本地，推送才会上传到GitHub

### Q3: 推送时提示认证失败？
**A**: 
1. 检查网络连接
2. 在终端运行: `git push origin main`
3. 如果提示输入密码，使用GitHub Personal Access Token

### Q4: 如何撤销提交？
**A**: 
- 点击 `...` 菜单
- 选择"撤销上次提交"
- 或在终端运行: `git reset --soft HEAD~1`

### Q5: 如何查看提交历史？
**A**: 
- 点击源代码管理面板顶部的"时钟"图标
- 或点击 `...` → "查看历史"
