# 列宽调整手柄样式优化

## 问题
原有的列宽调整手柄样式过于明显，影响表格美观度。

## 解决方案
参考互联网主流设计模式，采用"隐藏式手柄 + 悬停提示"的方案：

### 优化内容
1. **隐藏默认手柄视觉样式**
   - 将手柄背景设为透明
   - 移除默认的边框和图标
   - 保留功能区域（10px 宽度）和鼠标指针提示

2. **添加悬停视觉反馈**
   - 鼠标悬停在表头单元格时，在列边界显示蓝色细线提示
   - 拖拽时加强视觉反馈（颜色加深）
   - 使用平滑过渡动画

### 技术实现
- 修改 `ResizableTable.tsx` 中手柄的 inline style
- 新增 `ResizableTable.css` 自定义样式文件
- 使用 CSS 伪元素 `::after` 实现悬停提示

### 功能影响
✅ **不影响功能** - 列宽调整功能完全保留：
- 鼠标移到列边界时仍显示 `col-resize` 光标
- 拖拽调整列宽功能正常工作
- 阻尼效果和最小宽度限制保持不变
- localStorage 存储列宽设置正常

### 用户体验提升
- 表格外观更简洁美观
- 仅在需要时显示调整提示
- 符合现代 UI 设计规范
- 与 Ant Design 风格更协调

## 测试方法
1. 启动应用，进入发布任务页面
2. 观察表格表头，默认状态下看不到调整手柄
3. 鼠标悬停在表头单元格上，右侧边界会显示蓝色细线
4. 在列边界处拖拽，可以正常调整列宽
5. 刷新页面，列宽设置被正确保存

## 参考资料
根据互联网最佳实践，主流表格组件（如 TanStack Table、Material-UI Table、shadcn/ui）都采用类似的隐藏式手柄设计，仅在交互时提供视觉反馈。
