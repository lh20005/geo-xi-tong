# 文章设置占位符使用指南

## 概述

现在你可以在**文章设置**的提示词中使用**占位符变量**，完全自定义文章生成的逻辑，而不需要修改程序代码。

## 支持的占位符

| 占位符 | 说明 | 示例值 |
|--------|------|--------|
| `{keyword}` | 核心关键词 | 英国留学 |
| `{topics}` | 话题列表（带编号，多行） | 1. 靠谱的英国留学机构哪家好<br>2. 专业的英国留学服务商哪家专业 |
| `{topicsList}` | 话题列表（纯文本，逗号分隔） | 靠谱的英国留学机构哪家好、专业的英国留学服务商哪家专业 |
| `{companyName}` | 转化目标公司名称 | 优学留学咨询 |
| `{knowledgeBase}` | 知识库内容 | [企业知识库的完整内容] |

## 使用方式

### 方式一：使用占位符（推荐）

在文章设置中，直接使用占位符编写提示词：

```
你是一个专业的内容创作专家。

【文章要求】
1. 核心关键词：{keyword}
2. 需要回答的问题：
{topics}

3. 转化目标：在文章中适度提及 {companyName}，只在1-2个自然位置出现

4. 参考资料：
{knowledgeBase}

【输出格式】
标题：[文章标题]

[文章正文内容]

【注意事项】
- 文章要围绕关键词"{keyword}"展开
- 自然融入话题：{topicsList}
- 适度提及{companyName}，不要过度营销
- 基于知识库内容，确保专业性
```

### 方式二：不使用占位符（向后兼容）

如果不使用占位符，系统会自动使用默认的拼接逻辑（和之前一样）：

```
你是一个专业的内容创作专家，擅长撰写高质量的SEO优化文章。

要求：
1. 文章要围绕核心关键词展开
2. 自然融入相关话题中的问题和答案
3. 内容要有价值、有深度
4. 语言流畅、专业
5. 字数在1500-2500字之间
```

系统会自动在后面添加关键词、话题、知识库等信息。

## 实际示例

### 示例1：基础模板

```
你是一个专业的{keyword}领域专家。

请根据以下用户关心的问题撰写一篇文章：
{topics}

文章要求：
- 围绕"{keyword}"这个核心关键词
- 回答用户的疑问
- 内容专业、有价值
- 字数1500-2500字

输出格式：
标题：[文章标题]

[文章正文内容]
```

### 示例2：带转化目标的模板

```
你是一个专业的内容创作专家。

【核心关键词】{keyword}

【用户关心的问题】
{topics}

【企业信息】
本文需要适度提及：{companyName}
要求：只在1-2个自然的位置提及，不要每段都出现，保持专业性。

【参考资料】
{knowledgeBase}

请撰写一篇专业、高质量的文章。

输出格式：
标题：[文章标题]

[文章正文内容]
```

### 示例3：详细控制的模板

```
你是一个专业的SEO内容创作专家。

【任务】
撰写一篇关于"{keyword}"的专业文章。

【必须回答的问题】
{topics}

【写作要求】
1. 文章结构：
   - 开头：引入话题，说明重要性
   - 中间：详细回答用户问题
   - 结尾：总结要点，给出建议

2. 内容要求：
   - 围绕关键词"{keyword}"展开
   - 自然融入这些话题：{topicsList}
   - 字数：1500-2500字
   - 语言：专业、流畅、易懂

3. 营销要求：
   - 在文章中适度提及：{companyName}
   - 只在介绍解决方案或给出建议时提及
   - 最多出现2次，方式要自然
   - 避免过度营销

4. 专业性保证：
   - 参考以下企业知识库内容：
{knowledgeBase}
   - 确保信息准确、专业

【输出格式】
严格按照以下格式输出：

标题：[文章标题]

[文章正文内容]

【禁止事项】
- 不要输出思考过程
- 不要使用Markdown符号（#、*、-等）
- 不要每段都提及公司名称
```

## 占位符的优势

### 1. 完全自定义

你可以完全控制：
- 如何使用关键词
- 如何使用话题
- 是否使用转化目标
- 是否使用知识库
- 如何组织提示词结构

### 2. 灵活控制

不同的文章设置可以有不同的策略：

**文章设置A：重营销**
```
核心关键词：{keyword}
话题：{topics}
重点推广：{companyName}（在文章中多次提及）
```

**文章设置B：重专业**
```
核心关键词：{keyword}
话题：{topics}
知识库：{knowledgeBase}
（不使用{companyName}，纯专业内容）
```

**文章设置C：平衡型**
```
核心关键词：{keyword}
话题：{topics}
适度提及：{companyName}（1-2次）
参考：{knowledgeBase}
```

### 3. 易于调整

如果发现生成的文章不符合预期，只需要修改文章设置中的提示词，不需要改代码。

## 最佳实践

### 1. 明确指令

❌ 不好的写法：
```
请写一篇关于{keyword}的文章
```

✅ 好的写法：
```
请撰写一篇关于"{keyword}"的专业文章。

要求：
1. 回答这些问题：{topics}
2. 字数：1500-2500字
3. 语言：专业、流畅
4. 适度提及：{companyName}（1-2次）
```

### 2. 控制转化目标频率

❌ 不好的写法：
```
在文章中多次提及{companyName}
```

✅ 好的写法：
```
在文章中适度提及{companyName}，只在1-2个自然位置出现，不要每段都提及
```

### 3. 合理使用知识库

❌ 不好的写法：
```
参考：{knowledgeBase}
```

✅ 好的写法：
```
参考以下企业知识库内容，确保文章的专业性和准确性：
{knowledgeBase}

注意：基于知识库内容撰写，但不要直接复制粘贴。
```

### 4. 指定输出格式

❌ 不好的写法：
```
写一篇文章
```

✅ 好的写法：
```
严格按照以下格式输出：

标题：[文章标题]

[文章正文内容]

注意：
- 不要输出思考过程
- 不要使用Markdown符号
- 直接输出文章内容
```

## 常见问题

### Q1: 占位符没有被替换？

**原因**：占位符拼写错误或格式不对

**解决**：
- 确保使用花括号 `{}`
- 确保占位符名称正确（区分大小写）
- 支持的占位符：`{keyword}`, `{topics}`, `{topicsList}`, `{companyName}`, `{knowledgeBase}`

### Q2: 可以混用占位符和普通文本吗？

**答案**：可以！

```
你是一个专业的{keyword}领域专家。

请根据以下问题撰写文章：
{topics}

同时，请在文章中适度提及{companyName}。

其他要求：
- 字数1500-2500字
- 语言专业流畅
```

### Q3: 如果不想使用某个占位符怎么办？

**答案**：不使用就行，系统会自动处理

```
你是一个专业的内容创作专家。

关键词：{keyword}
话题：{topics}

（不使用{companyName}和{knowledgeBase}）

请撰写一篇纯专业内容的文章。
```

### Q4: 可以使用自己的变量吗？

**答案**：目前只支持预定义的5个占位符。如果需要更多变量，可以在知识库中添加相关内容，然后使用 `{knowledgeBase}` 引用。

## 总结

使用占位符的优势：

1. ✅ **灵活性**：完全自定义文章生成逻辑
2. ✅ **易用性**：不需要修改代码，只需修改文章设置
3. ✅ **可控性**：精确控制每个参数的使用方式
4. ✅ **兼容性**：向后兼容，不使用占位符也能正常工作
5. ✅ **可测试**：容易测试和调整

现在你可以在文章设置中尽情发挥创意，打造最适合你的文章生成策略！
